As for Perl just download the usual Win package and run the
installer.<br><br>
As for RealVNC, dowload and install the full package only on the server
and use the viewer only on your work station to maintain some kind of
security.<br>
Open port 5800 and 5900 for your router to allow RealVNC access
anywhere.<br><br>
Also note, if you try to manage the Xitami from RealVNC, you will
probably need to add a short cut to the folder for Xitami and one for
your web directory paths if you use the virtual hosts in a separate
folder. You will also have to right click the folders and other shortcuts
and select OPEN rather than double click it when you use the viewer,
however it seems to work ok in the web viewer version.<br><br>
To log into and admin your Xitami you can then disable your web admin to
protect xitami better, and also edit the defaults.cfg file
manually.<br><br>
A very helpful hint to use in all this is to add shortcuts to notepad in
the strategical places like the xitami root foler and your cfg files
path.<br>
Then you can drag and drop files there to edit rather than assign generic
extension tags to allow windows to control each file extension.<br><br>
Further, to open the web viewer of RealVNC in this, navigate to your IP
address ###.###.###.###:5800 to log in and you're set.<br><br>

Again go read up on the terms so this makes sense and you don't spin
your<br>
wheels.&nbsp; The term &quot;modem&quot; (as in DSL Modem or similarly
Cable Modem) is most<br>
likely a router; in fact you might also refer to it as a
&quot;bridge&quot; (in a<br>
looser sense but that is off topic).&nbsp; The modem establishes the
connection<br>
to your ISP and then converts it to route into your home
environment.&nbsp; This<br>
device usually will NAT the external IP into an internal IP (like<br>
169.254.xxx.xxx) and it may be programmed to allow you something like
four<br>
machines to simultaneously go through it to the internet.&nbsp; This
means it can<br>
assign up to four addresses internally.&nbsp; The router in this example
is a<br>
mini DHCP server or sorts.&nbsp; If so then it is routing four internal
IP<br>
addresses through a single NAT'ed external IP address (it could also
be<br>
using port address translation too, PAT).&nbsp; The point is don't get
all in a<br>
bunch over semantics; do your own homework and then ask some
knowledgeable<br>
questions.<br><br>
Good luck,<br>
